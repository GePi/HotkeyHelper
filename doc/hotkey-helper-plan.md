# HotKey Helper — План разработки

## Концепция
Desktop-приложение для Windows 11, которое постоянно отображается на отдельном мониторе и автоматически показывает список горячих клавиш для приложения, находящегося в фокусе.

## Стек технологий
| Компонент         | Технология                  |
|-------------------|-----------------------------|
| Язык              | Java 17+ (LTS)              |
| UI-фреймворк      | JavaFX (OpenJFX 21+)        |
| Доступ к WinAPI   | JNA 5.x                     |
| Парсинг данных    | Gson 2.x                    |
| Сборка            | Gradle (Kotlin DSL)         |
| Упаковка в .exe   | jpackage через плагин jlink |

## Фазы разработки

---

### Фаза 1 — Каркас проекта
**Цель:** пустое JavaFX-окно запускается и отображается.

**Задачи:**
1. Создать Gradle-проект (`build.gradle.kts`) с зависимостями (JavaFX, JNA, Gson)
   - Плагины: `org.openjfx.javafxplugin`, `application`, `org.beryx.jlink`
2. Настроить `module-info.java` (JavaFX требует модули)
3. Реализовать `App.java` — минимальный JavaFX `Application`
4. Окно без рамки (`StageStyle.UNDECORATED`), always on top
5. Позиционирование окна на втором мониторе через `Screen.getScreens()`
6. Проверить запуск и отображение

**Результат:** пустое тёмное окно на втором мониторе.

---

### Фаза 2 — Отслеживание активного окна
**Цель:** приложение знает, какой процесс сейчас в фокусе.

**Задачи:**
1. Реализовать `WindowTracker.java`:
   - JNA-интерфейс к `User32.dll`: `GetForegroundWindow()`
   - JNA-интерфейс к `Kernel32.dll` / `Psapi.dll`: получение имени процесса
   - Цепочка вызовов: `GetForegroundWindow()` → `GetWindowThreadProcessId()` → `OpenProcess()` → `GetModuleFileNameEx()` → извлечь имя .exe
2. Фоновый `ScheduledExecutorService` с интервалом 300 мс
3. Колбэк `Consumer<String>` вызывается только при смене процесса (дедупликация)
4. Вывод имени процесса в `Label` на экране для отладки

**Результат:** окно показывает текст вроде "chrome.exe" / "code.exe" при переключении окон.

---

### Фаза 3 — База горячих клавиш
**Цель:** загрузка и поиск горячих клавиш по имени процесса.

**Задачи:**
1. Определить формат `shortcuts.json`:
   ```json
   {
     "apps": {
       "chrome.exe": {
         "name": "Google Chrome",
         "icon": "chrome.png",
         "groups": [
           {
             "title": "Вкладки",
             "shortcuts": [
               { "keys": "Ctrl+T", "description": "Новая вкладка" },
               { "keys": "Ctrl+W", "description": "Закрыть вкладку" },
               { "keys": "Ctrl+Tab", "description": "Следующая вкладка" }
             ]
           },
           {
             "title": "Навигация",
             "shortcuts": [
               { "keys": "Alt+←", "description": "Назад" },
               { "keys": "Ctrl+L", "description": "Фокус на адресную строку" }
             ]
           }
         ]
       }
     }
   }
   ```
2. Реализовать `ShortcutRepository.java`:
   - Загрузка JSON при старте через Gson
   - Метод `getShortcuts(String processName) → AppShortcuts`
   - Hot-reload: `WatchService` на файл, перезагрузка при изменении
3. Создать начальный файл с 2-3 приложениями для тестирования

**Результат:** по имени процесса можно получить структурированный список клавиш.

---

### Фаза 4 — Отображение горячих клавиш (основной UI)
**Цель:** при переключении окон на экране появляется список клавиш.

**Задачи:**
1. Реализовать `MainView.java`:
   - Заголовок с именем приложения
   - Группы горячих клавиш с подзаголовками
   - Каждая клавиша: комбинация (выделена) + описание
   - `ScrollPane` для длинных списков
2. Подключить CSS-тему:
   - Тёмный фон (#1e1e2e или подобный)
   - Светлый текст
   - Акцентный цвет для комбинаций клавиш
   - Моноширинный шрифт для клавиш
3. Сообщение-заглушка, если для процесса нет данных: "Нет горячих клавиш для {process}"

**Результат:** полностью рабочий прототип — переключаешь окна, видишь клавиши.

---

### Фаза 5 — Полировка и удобства
**Цель:** довести до ежедневного использования.

**Задачи:**
1. Трей-иконка (SystemTray) с меню:
   - Выбор монитора
   - Открыть shortcuts.json для редактирования
   - Выход
2. Запоминание позиции окна между запусками (сохранение в config.json)
3. Перетаскивание окна за кастомный title bar
4. Строка поиска / фильтр по горячим клавишам (Ctrl+F)
5. Обработка edge-cases:
   - Второй монитор отключён — показать на основном
   - UWP-приложения (ApplicationFrameHost.exe) — определение реального приложения
   - Окно без процесса (рабочий стол, панель задач)

**Результат:** приложение комфортно для повседневного использования.

---

### Фаза 6 — Сборка и дистрибуция
**Цель:** .exe файл, который можно просто запустить.

**Задачи:**
1. Настроить `org.beryx.jlink` плагин для сборки в .exe/.msi:
   - Встроенная JRE (пользователю не нужна Java)
   - Иконка приложения
   - Автозапуск при входе в Windows (опционально)
2. Тестирование на чистой системе
3. README с инструкцией по формату shortcuts.json

**Результат:** готовый дистрибутив.

## Риски и как их снизить

| Риск | Решение |
|------|---------|
| JNA не видит процесс (права доступа) | Запуск с повышенными правами или фильтрация ошибок |
| UWP-приложения скрыты за ApplicationFrameHost.exe | Дополнительный WinAPI-вызов для определения реального пакета |
| JavaFX тяжёлый при старте (~2-3 сек) | Splash screen или скрытый старт в трее |
| Пользователь не знает формат JSON | Пример файла + валидация с понятными ошибками |
