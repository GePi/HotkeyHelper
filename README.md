# HotKey Helper

Desktop-приложение для Windows, которое автоматически показывает горячие клавиши для приложения, находящегося в фокусе. Предназначено для работы на втором мониторе — переключаете окна, а на экране всегда актуальная шпаргалка.

## Возможности

- Автоматическое определение активного окна (процесс + заголовок)
- Отображение горячих клавиш в виде HTML-страниц с тёмной темой
- Фильтрация по имени процесса и заголовку окна (поддержка регулярных выражений)
- Позиционирование на втором мониторе
- Изменение размера окна, стандартные кнопки управления окном
- Статус-бар с информацией о текущем процессе и заголовке окна

## Требования для запуска

- Windows 10/11
- Java на машине **не требуется** — JRE встроена в сборку

## Требования для сборки

- JDK 21+
- Gradle 8.5+
- [WiX Toolset](https://wixtoolset.org/) не требуется (собираем portable, не установщик)

## Сборка и запуск

```bash
# Запуск из исходников (нужна JDK 21+)
./gradlew run

# Сборка portable .exe (со встроенной JRE, Java на целевой машине не нужна)
./gradlew jpackage

# Результат: build/jpackage/HotkeyHelper/
#   HotkeyHelper.exe    — исполняемый файл
#   runtime/             — встроенная JRE
#   app/                 — модули приложения
```

### Портабельная версия

После `./gradlew jpackage` папка `build/jpackage/HotkeyHelper/` — это готовая portable-версия. Скопируйте её вместе с папкой `data/` в любое место и запускайте `HotkeyHelper.exe`. Установка и Java на машине не нужны.

```
HotkeyHelper/                  # Скопировать куда угодно
├── HotkeyHelper.exe           # Запускаемый файл

└── data/                      # Конфигурация (положить рядом)
    ├── config.json
    └── shortcuts/

        └── ...
```

## Конфигурация

Настройка приложений выполняется в файле `data/config.json`. Подробная документация по формату конфигурации и созданию HTML-файлов — в [data/README.md](data/README.md).

### Быстрый пример

```json
{
  "apps": [
    {
      "processName": "chrome.exe",
      "windowTitle": null,
      "htmlFile": "chrome.html"
    },
    {
      "processName": "1cv8.exe",
      "windowTitle": "(Конфигуратор|Отладка)",
      "htmlFile": "1c.html"
    }
  ]
}
```

- `processName` — имя процесса (регистронезависимое)
- `windowTitle` — регулярное выражение для фильтрации по заголовку окна (`null` = любой заголовок)
- `htmlFile` — HTML-файл из папки `data/shortcuts/`

## Архитектура

```
┌──────────────────────────────────────────────┐
│                  App.java                     │
│          (JavaFX Application, UI)             │
│                                               │
│  ┌─────────────────┐  ┌───────────────────┐  │
│  │ WindowTracker    │  │ WebView           │  │
│  │ (JNA, 300ms poll)│──│ (HTML shortcuts)  │  │
│  └────────┬────────┘  └───────────────────┘  │
│           │                     ▲             │
│           ▼                     │             │
│  ┌─────────────────────────────┘             │
│  │ ShortcutRepository                        │
│  │ (config.json + HTML files)                │
│  └───────────────────────────────────────────┘
└──────────────────────────────────────────────┘
```

1. **WindowTracker** каждые 300 мс через JNA опрашивает Windows API (`GetForegroundWindow` -> `GetWindowThreadProcessId` -> `QueryFullProcessImageNameW`) и определяет активный процесс и заголовок окна.
2. При смене окна вызывается колбэк, который запрашивает у **ShortcutRepository** HTML-файл, соответствующий текущему процессу.
3. **ShortcutRepository** ищет совпадение в `config.json` по имени процесса и (опционально) regex-паттерну заголовка окна.
4. Найденный HTML загружается в **WebView** с тёмной CSS-темой.

## Добавление нового приложения

1. Узнайте имя процесса — оно отображается в статус-баре приложения.
2. Создайте HTML-файл в `data/shortcuts/` по шаблону из [data/README.md](data/README.md).
3. Добавьте запись в `data/config.json`.
4. Перезапустите приложение.
